Developer note: This form persists to localStorage under the key medihelp_users so the demo works with zero backend, and sets medihelp_current_user as a simple session pointer. index.html reads these same keys on load to populate the Medical ID card and SOS emergency contact. Before real users touch this, replace saveUser() with a POST to your own /api/register endpoint, run over HTTPS, with server-side password hashing (bcrypt/argon2) and a real database (Postgres/MySQL) instead of the browser store.
